= Mounting a git credetials store into a workspace container

This section describes how to mount git credetials store as secret from the user's {orch-namespace} into the file in single-workspace or multiple-workspace containers of {prod-short}.

.Prerequisites

* A running instance of {prod}. To install an instance of {prod}, see {link-installing-an-instance}.

.Procedure
. Prepare git credential file in format https://git-scm.com/docs/git-credential-store#_storage_format.
. Encode content of the file to the base64 format.
. Create a new Kubernetes secret in the Kubernetes {orch-namespace} where a {prod-short} workspace will be created.

* The labels of the secret that is about to be created must match the set of labels configured in `che.workspace.provision.secret.labels` property of {prod-short}. The default labels are:

* `app.kubernetes.io/part-of: che.eclipse.org` 
* `app.kubernetes.io/component: workspace-secret`:
